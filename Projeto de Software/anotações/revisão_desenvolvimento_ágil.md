# DESENVOLVIMENTO ÁGIL
- ...
- equipe ágil ➔ capaz de responder apropiadamente a mudanças (no sofwtare, na equipe, nas tecnologias)
  - as mudanças durante o desenvolvimento do software aumentam os custos de forma não linear conforme o projeto avança
- processo ágil de software deve se adaptar incrementalmente
  - para essa adptação a equipe precisa so feedback do cliente ➔ para esse feedback ocorre é preciso entregar para o cliente um protótipo operacional ou partes do software em curtos períodos de tempo ➔ as adpatções devem acompanhar o mesmo ritmo das mudanças (imprevisibilidade)
  - para que ocorra essas entregas o software deve ter um desenvolvimento incremental

### Modelos de PROCESSOS ÁGEIS
- Extreme Programming - XP
- Scrum
- Método de desenvolvimento de sistemas dinâmicos - DSDM
- Desenvolvimento dirigido a Funcionalidade - FDD

## Extreme Programming - XP
- emprega abordagem orientada a objetos
- conjunto de regras e práticas constantes no contexto de 4 atividades metodológicas: planejamento, projeto, codificação e testes.

- planejamento
  - levantamento de requisitos ➔ entender o ambiente de negócios do software, desenvolver uma percepção ampla sobre os resultados solicitados, fatores principais e funcionalidades

  - princípio KIS - keep it simple  ➔ preserva a simplicidade

  - solução pontuais protótipos
- codificação
- testes
